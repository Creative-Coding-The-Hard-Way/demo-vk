3/// Criterion when to rotate the log file.
4///
5/// Used in [`Logger::rotate`](crate::Logger::rotate).
6#[derive(Copy, Clone, Debug)]
7pub enum Criterion {
8    /// Rotate the log file when it exceeds the specified size in bytes.
9    Size(u64),
10    /// Rotate the log file when it has become older than the specified age.
11    ///
12    /// ## Minor limitation
13    ///
14    /// ### TL,DR
15    /// the combination of `Logger::append()`
16    /// with `Criterion::Age` works OK, but not perfectly correct on Windows or unix
17    /// when the program is restarted.
18    ///
19    /// ### Details
20    /// Applying the age criterion works fine while your program is running.
21    /// Ideally, we should also apply it to the rCURRENT file when the program is restarted
22    /// and you chose the `Logger::append()` option.
23    ///
24    /// Unfortunately, this does not work on Windows, and it does not work on unix,
25    /// for different reasons.
26    ///
27    /// To minimize the impact on age-based file-rotation,
28    /// `flexi_logger` uses on Windows, and on all other platforms where the creation date
29    /// of a file is not available (like on Unix), the last modification date
30    /// (or, if this is also not available, the current time stamp)
31    /// as the created_at-info of an rCURRENT file that already exists when the program is started,
32    /// and the current timestamp when file rotation happens during further execution.
33    /// Consequently, a left-over rCURRENT file from a previous program run will look newer
34    /// than it is, and will be used longer than it should be.
35    ///
36    /// #### Issue on Linux
37    ///
38    /// Linux does not maintain a created-at property for files, only a last-changed-at property.
39    ///
40    /// #### Issue on Windows
41    ///
42    /// For compatibility with DOS (sic!), Windows magically transfers the created_at-info
43    /// of a file that is deleted (or renamed) to its successor,
44    /// when the recreation happens within some seconds [\[1\]](#ref-1).
45    ///
46    /// If the file property were used by `flexi_logger`,
47    /// the rCURRENT file would always appear to be as old as the
48    /// first one that ever was created - rotation by time would fail completely.
49    ///
50    /// <a name="ref-1">\[1\]</a> [https://superuser.com/questions/966490/windows-7-what-is-date-created-file-property-referring-to](https://superuser.com/questions/966490/windows-7-what-is-date-created-file-property-referring-to).
51    ///
52    Age(Age),
53    /// Rotate the file when it has either become older than the specified age, or when it has
54    /// exceeded the specified size in bytes.
55    ///
56    /// See documentation for Age and Size.
57    AgeOrSize(Age, u64),
58}
