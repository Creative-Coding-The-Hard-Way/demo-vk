1//! Error codes of `flexi_logger`.
2//!
3//! The following error codes are used to indicate the reason of an error.
4//! More details on them can be found here.
5//!
6//! ## `Write`
7//!
8//! Writing the log line to the output failed.
9//!
10//! Example:
11//!
12//! ```text
13//! [flexi_logger][ERRCODE::Write] writing log line failed, caused by Send
14//! ```
15//!
16//! Possible reasons depend on the `WriteMode` and the output channel.
17//!
18//! With an asynchronous `WriteMode`, the root cause can be that the logger handle that was returned
19//! from the logger initialization was not assigned to a variable to keep it alive (see also
20//! [`Logger::start()`](https://docs.rs/flexi_logger/latest/flexi_logger/struct.Logger.html#method.start)).
21//! It is then dropped immediately, and in its `Drop` impl it cleans up all resources,
22//! including the asynchronous writer. So the next log output will fail with this error.
23//!
24//! ## `Flush`
25//!
26//! Explicit or automatic flushing of buffered log lines to the output failed.
27//!
28//! Example:
29//!
30//! ```text
31//! [flexi_logger][ERRCODE::Flush] flushing primary writer failed, caused by Send
32//! ```
33//!
34//! For possible reasons, see [Write](#write).
35//!
36//! ## `BindToCore`
37//!
38//! Binding a thread to a specific CPU core failed.
39//!
40//! ## `Format`
41//!
42//! The chosen format function had produced an error.
43//!
44//! Example:
45//!
46//! ```text
47//! [flexi_logger][ERRCODE::Format] formatting failed, caused by ...
48//! ```
49//!
50//! If this happens with one of `flexi_logger`s provided format functions, please open an issue.
51//!
52//! ## `LogFile`
53//!
54//! The `FileLogWriter` is not able to rotate the log file. The reason should be printed as well.
55//!
56//! ## `LogFileWatcher`
57//!
58//! The `FileLogWriter` is not able to watch the log file. The reason should be printed as well.
59//!
60//! ## `LogSpecFile`
61//!
62//! This error can only occur if you use `Logger::start_with_specfile`, where you specify a
63//! log-specification-file that you can edit, while the program is running, to influence
64//! which log lines it should write.
65//!
66//! Examples:
67//!
68//! ```text
69//! [flexi_logger][ERRCODE::LogSpecFile] continuing with previous log specification,
70//! because rereading the log specification file failed, caused by ...
71//! ```
72//!
73//! The log-specification-file you chose with `Logger::start_with_specfile` cannot be opened,
74//! read, or successfully parsed.
75//!
76//! ```text
77//! [flexi_logger][ERRCODE::LogSpecFile] error while watching the specfile, caused by ...
78//! ```
79//!
80//! Watching the log-specification-file failed.
81//!
82//! ## `Poison`
83//!
84//! Log entries can be written by all threads of your program. Loggers thus must be thread-safe,
85//! by guarding their mutable parts with `Mutex`es, `RwLocks`, etc. In case that a thread panics
86//! while owning one of these locks, the lock is subsequently considered "poisoned".
87//!
88//! A typical root cause for this is some `panic!` in a `Debug` or `Display` implementation
89//! of a logged object.
90//!
91//! ## `Symlink`
92//!
93//! This error can only occur on unix systems, and when you use `Logger::create_symlink`, and
94//! indicates an issue with creating or replacing the symbolic link to the log file.
95//!
96//! ## `WriterSpec`
97//!
98//! The code uses in some log macro call the syntax to send the log line to a certain `LogWriter`,
99//! but this log writer does not exist.
100//!
